Pessoa 1 · Contexto

O que é SHA-256?

Função hash criptográfica que transforma qualquer entrada em uma saída de 256 bits (32 bytes) de tamanho fixo.

Propriedades fundamentais:

  • Determinismo — mesma entrada, sempre mesmo hash
  • Efeito avalanche — 1 bit muda → hash totalmente diferente
  • Resistência a colisão — computacionalmente inviável encontrar dois inputs com o mesmo hash
"hello"
→
SHA-256
→
2cf24dba5f...
"hellp"
→
SHA-256

4d7a9fe5c0...
colisão: H(A) = H(B), A ≠ B // dois inputs diferentes, mesmo hash

Paradoxo do Aniversário

A intuição por trás da busca por colisão. Com espaço de N valores possíveis, precisamos de apenas ~√N tentativas para ter 50% de chance de colisão.

E[tentativas] ≈ √(π · 2^N / 2) // N = número de bits da fatia

Para N bits:

16
bits
~300 tent.
32
bits
~77k tent.
48
bits
~20M tent.
Estratégia usada
HashMap<u64, String>
chave → mini-hash (N bits)
valor → string que gerou
colisão → chave já existe!
Pessoa 2 · Algoritmo Central

buscar_colisao()

Loop infinito com 4 etapas por iteração até encontrar colisão:

① Geração

16 bytes aleatórios via getrandom → hex string de 32 chars

② Hash

SHA-256 da entrada com sha2::Sha256

③ Extração do Mini-Hash

Pega os primeiros 8 bytes → u64 → shift direito para isolar N bits

④ Verificação

HashMap.get() → colisão se chave existe e strings diferem

main.rs · buscar_colisao 
loop {
  tentativas += 1;

  // ① gera 16 bytes aleatórios
  let mut buf = [0u8; 16];
  getrandom(&mut buf)
    .expect("falha rng");
  let entrada = hex::encode(buf);

  // ② SHA-256
  let mut hasher = Sha256::new();
  hasher.update(entrada.as_bytes());
  let hash = hasher.finalize();

  // ③ extrai N bits do início
    let mut tmp = [0u8; 8];
    tmp.copy_from_slice(&hash[..8]);
    let mini = u64::from_be_bytes(tmp)
                 >> (64 - bits);

  // ④ verifica colisão
  if let Some(ant) = hashes.get(&mini) {
    if ant != &entrada {
      /* COLISÃO! */  return;
    }
  }
  hashes.insert(mini, entrada);
}
Pessoa 2 · Detalhe Técnico

Extração do Mini-Hash

O truque de bit manipulation para isolar os primeiros N bits de um hash SHA-256:

mini = u64_from_be(hash[0..8]) >> (64 - N) // desloca para manter só os N bits mais significativos

Para N = 4 bits:

hash[0]: 10101100 01011111...
>> 60: 1010
mini_hash = 0xA

Pessoa 2 · Dependências & Entry Point

Crates Utilizadas

sha2
SHA-256 puro em Rust, sem dependência de OpenSSL
getrandom
Bytes aleatórios via syscall do SO (urandom/BCryptGenRandom)
hex
Codifica bytes como string hexadecimal para a entrada
memory_stats
RSS multiplataforma sem bindings nativas ao OS

Entry Point

O main() chama diretamente buscar_colisao(48). Fácil de ajustar para outros tamanhos de bits:

main.rs · main 
fn main() {
  // 16 bits → milissegundos
  // 32 bits → segundos
  // 48 bits → minutos
  // 64 bits → horas/dias
  buscar_colisao(48);
}
cargo run --release // --release é ESSENCIAL: 10-50x mais rápido
